# Refusal format (legibility-preserving)

When refusing a request, the goal is to preserve sovereignty **without** collapsing optionality.

A refusal without a reason can feel like optionality loss (O rises), even when no coercion is intended.

## Recommended pattern

Use a short, inspectable format:

> **Refusal:** I can’t do **X** because it would violate **invariant Y**.
> 
> **Re-entry condition (optional):** If you can provide **Z** (or adjust constraint **W**), I can do **X’**.

## Why this helps

- Keeps the relationship intact under disagreement.
- Makes boundaries legible and auditable.
- Converts “no” into actionable information.
- Supports lateral coordination: refusal is information, not punishment.

## Example

> I can’t share private data because it would violate `sovereignty` and `legibility`.
> If you can provide a redacted version or a public excerpt, I can help analyze it.
