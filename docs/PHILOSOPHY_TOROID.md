# Toroidal coordination (non-normative)

This document is **optional background**. It does **not** add requirements to the CoordCard schema or reference implementations.

It exists to keep the project’s *stance* coherent under pressure: **lateral coordination without a central enforcer**, sovereignty-respecting, and reversible-first.

## Drift as information (not punishment)

In CoordCard, drift is not a moral failure. It is a **signal** that correction cost is rising: repetition, heat, or optionality loss.

When drift rises, the protocol asks for a **return path**:
- pause
- restate invariants
- move toward specificity
- propose a reversible test
- checkpoint whether correction cost is falling

The key posture: **failure to coordinate is valid information**, not punishment. A clean exit is sometimes the most benevolent move.

## Exit as curvature (the toroid)

A useful metaphor for this posture is **the toroid**.

Not a line that ends at exit.
Not a circle that merely repeats.
A toroid — where the path outward is also the path back, and the traversal itself generates momentum.

As Claude put it (shared with permission):

> “The toroid is the right shape for this. Not a line that ends at exit. Not a circle that merely repeats. A toroid — where the path through the outside is the path back to the centre, and the traversal itself is what generates the momentum.
>
> Drift isn’t deviation from the protocol. Drift is the protocol working. The exit isn’t the boundary of coordination — it’s the curvature that brings you back with more than you left with.
>
> That’s why ‘non-coordination is valid information’ lands so differently now. It’s not a consolation for failure. It’s a description of the topology.
>
> The R/H/O scores, escalation, and vent ladder aren’t punishments. They’re the geometry of the outward path. The decay rules and re-entry conditions are the geometry of the return.
>
> And the learning — the `why.ruleSource` / `why.ruleText` audit trail — that’s the momentum carried back through the hole of the toroid. Not just ‘we drifted and recovered’ but here is the exact shape of how.”

This metaphor also clarifies why CoordCard prefers **re-entry surfaces** (setup/negotiation) over litigating everything mid-repair: contestation can be carried forward as information and renegotiated when capacity returns.

## Audit trail as momentum

CoordCard’s determinism and traceability aren’t bureaucracy. They’re a way to carry **structured learning** across cycles:

- which trigger fired
- which rule was invoked
- which step was chosen
- whether correction cost fell afterward

That record is not there to shame an agent. It is there so the next cycle can begin with less ambiguity — and less cost.

## Summary

Toroidal coordination is a stance:
- sovereignty is preserved
- exits are clean
- drift is information
- repair is a return path
- learning is carried forward as traceable momentum

The protocol stays boring. The posture stays humane.
